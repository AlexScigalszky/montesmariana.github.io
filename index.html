On October 12th I added the latest (hopefully last) sense-coding of church to the church clouds, as well as extra variables that might code extra context features (definiteness, modifiers) or sense-dimensions (building, community, organization...).

Click to show description of current clouds.

These clouds represent the similarity between tokens of state, country and church, either with all types at once, one at a time or only two combined.

The tokens come from the Semcor corpus. The ppmi values for first order weighting come from Semcor as well. The threshold indicated in the columns of the tables was applied to this weighting. After applying the threshold, tokens that had lost all the weighted context words were dropped.

The second-order co-occurrence (SOCC) ppmi values come from either Semcor or COHA, and it's indicated as the name of the link to the cloud. In all cases the same context words were used, only the ppmi values differ (and, apparently, not that much either). These are all the 5795 content words with frequency over 10 in both corpora.

The caption of the tables (for now only in Church) takes to to clouds with a ppmi threshold on the SOCC matrix.

The newBroadGenre category is between informative and imaginative, done by the compilers of the corpus themselves. It's well identified by the stylo R package. The broadGenre category is actually based on that (but I will adapt the names when I'm fixed on some option.) imag matches imaginative, tech includes the most technical texts (skill_and_hobbies, miscellaneous and learned), while the rest goes to the very practical others. It doesn't look that bad with stylo either.

A further problem is the imperfect pos-tagging. A corrected version can be seen as new_pos variable.

Now you're ready to enjoy the clouds. Please beware of saturation risk by looking at them for too long. Belgium has plenty of real clouds if you need to refresh your eyes.

Here you can look at some metadata about the clouds that are split by genre, generated on September 07 2018.

These are (perfectly safe) csv files with the following information:

  • Filename of the dataframe.
  • Type included in the cloud.
  • Corpus used for the SOCC matrix (either Semcor or COHA, with an i ending meaning that the intersect was used for the features).
  • The subsetting applied: either none, nona (=only semantically tagged tokens) or the name of the genre.
  • How many tokens are included.
  • The threshold applied to the first order weighting.
  • The stress value from the multidimensional scaling.
